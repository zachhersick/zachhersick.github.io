Birthday Paradox

The Birthday Paradox is a statistical phenomenon wherein if there are greater than 22 people in a room, the chance that at least two of them have the same birthday is greater than 50%. We wanted to test this, by writing a program that mimics it.

This project runs simulations where 20 trials are run of each multiple of 5 people up to 100, with each person having a randomly generated birthday. After all of the trials are run, it displays the percentage of trials for each increment of people that had duplicate birthdays.

This project has four classes, Birthday, Trial, Simulation, and Experiment. The Birthday class creates the Birthday objects, representing a person with a randomly generated birthday. The Trial class defines one trial in an experiment and checks for duplicate birthdays. The Experiment class runs the specified number of trials with the specified number of people per trial. Finally, the Simulation class brings everything together and runs the complete program.

I was responsible for writing and testing the Experiment and Simulation classes. While they were both fairly simple to code, with simple logic that ran the trials and experiments, respectfully, and returning the percentage of trials with duplicate birthdays in each experiment and the final table of data at the end of all of the experiments, the testing of experiment was specifically difficult. This was because the methods within experiment had random object instantiation, but it was used for 20 different size groups and 20 trials for each group. This was challenging, but I eventually realized that if I tested it making sure everyone had the same birthday, it would test the methods correctly. This project helped me shake off the rust of logic and solidify my knowledge of tesing.
